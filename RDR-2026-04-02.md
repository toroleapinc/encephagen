## 2. Experiments Conducted

### Experiment 10: Pendulum/Cart-pole Learning (FAILED)
- **Goal:** Get 1000 spiking neurons to learn pendulum balance
- **Approaches tried:** (1) Surrogate gradient + REINFORCE, (2) Frozen reservoir + trained readout, (3) Actor-Critic, (4) Rate-coded features, (5) Open-loop MLP baseline
- **Result:** ALL FAILED. Even the MLP baseline barely learned (200/500 steps on cart-pole).
- **Root cause:** Random spiking reservoirs don't provide useful state representations. REINFORCE has too high variance through spiking dynamics. Surrogate gradients through recurrent spiking cause exploding/vanishing gradients.
- **Key learning:** The problem was the RL approach, not the spiking network. Motor learning requires either engineered structure (Nengo/NEF) or specialized SNN training (PopSAN/e-prop).

### Experiment 11: Spontaneous Body Movement (6 iterations)
- **Goal:** What does a body DO when driven by spontaneous brain activity?
- **v1:** Motor silence (bad torque mapping)
- **v2:** Torques varied but identical across joints
- **v3:** Different brain regions → different joint torques (emergent somatotopy)
- **v4:** **Corrective tilt response emerged** (r=-0.43, brain corrects when body tilts)
- **v5:** Rhythmic motor activity at 44 Hz
- **v6:** All joints show rhythmic patterns (34-50 Hz)
- **Key finding:** Distributing motor control across different connectome regions naturally produces different motor commands for different joints. The brain produces weak corrective responses to body tilt purely from topology. Body falls after ~1s — torques too fast for body dynamics.

### Experiment 12-13: Brain + Spinal CPG + Body
- **Architecture:** Brain → tonic drive → Matsuoka CPG → alternating torques → MuJoCo body
- **CPG achievement:** 0.98 Hz alternating gait, left-right correlation = -1.000 (perfect anti-phase)
- **Matsuoka parameters (working):** tau=50, tau_adapt=500, c=2.0, w_mutual=2.5, w_crossed=1.5, beta=2.5
- **Brain modulation:** Sensory perturbation at 8-10s increased motor cortex drive by +0.024 → brain RESPONDS to stimuli and changes walking rhythm
- **Body survived 20 seconds** with brain + CPG architecture
- **Challenge:** Open-loop CPG locks into static state when body feedback is added. Proprioception stabilizes one CPG state instead of sustaining oscillation. Solved by using open-loop CPG (no body feedback to CPG).

### Experiment 14: Crawling Worm
- **Body:** 5-segment worm with traveling wave CPG
- **Result:** 0.31m forward displacement in 10 seconds
- **Abandoned:** User correctly noted the goal is a human brain, not a worm

### Experiment 15: Classical Conditioning
- **Architecture:** Visual cortex (CS) → connectome → amygdala (US/response). Three-factor STDP.
- **Protocol:** 30 pairings of visual stimulus with reward (amygdala activation)
- **Result:**
  - CS response before training: +0.00181
  - CS response after training: +0.00197 (+9% increase)
  - Novel stimulus response: +0.00075 (2.6x weaker than trained CS)
- **Significance:** Stimulus-specific learning. The brain learned that THIS visual pattern predicts reward.

### Experiment 16: Pattern Recognition
- **Protocol:** Present 5 distinct patterns to visual cortex, 5 times each. Classify from association cortex response.
- **Result:** 56% accuracy (chance = 20%, 2.8x above chance)
- **Analysis:** Cross-similarity 0.9994 (responses very similar but decodable). Leave-one-out nearest-centroid classifier.
- **Note:** No STDP training — this is purely from the connectome's differential routing of different input patterns.

### Experiment 17: Working Memory
- **Protocol:** Stimulus → visual cortex (0.5s) → remove → measure PFC persistence (2s delay)
- **Without NMDA:** 2% persistence (drops immediately)
- **With NMDA (tau=150ms):** 75% persistence at 500ms, 7% at 1.5s
- **Implementation:** Added slow NMDA synaptic current to PFC neurons only. Same synaptic input as AMPA but decays 30x slower. PFC recurrent connections sustain activity through NMDA.

### Experiment 18: Integrated Cognition
- **Protocol:** See pattern A → remember → pair with reward → test (A vs B response)
- **Result:** All four functions work together:
  - SEEING: ✓ (brain responds to visual input)
  - REMEMBERING: ✓ (PFC delay activity increases)
  - LEARNING: ✓ (trained stimulus response increases +0.00150)
  - SPECIFIC: ✓ (trained A grows more than control B, specificity +0.00025)

### Experiment 19: CPG Calibration Learning (BODY TOO STABLE)
- **Goal:** Brain learns to calibrate CPG drive for balance
- **Result:** SimpleBody with CPG never falls (body crouches to passively stable position)
- **Even with 50% torque noise:** 100% survival. Nothing to learn.
- **Root cause:** Body model lacks the instability needed for a learning task.

### Experiment 20: Walker2d Calibration
- **Body:** Gymnasium Walker2d (properly unstable, falls in <1s)
- **Result:**
  - Zero torques: 0.97s (falls)
  - Random torques: 0.16s (falls immediately)
  - Brain baseline (untrained): 1.22s (43% longer than zero!)
  - Brain after 30 learning episodes: 1.39s (+0.16s improvement)
- **Key finding 1:** The connectome's spontaneous activity produces better-than-nothing motor output (1.22s > 0.97s). The innate structure helps.
- **Key finding 2:** Reward-modulated STDP produces modest improvement (+0.16s). The baby is learning to calibrate, slowly.
- **Framework:** 先天程序 (innate CPG + connectome routing) × 后天学习 (reward-modulated STDP) = improved performance

## 3. Technical Additions

### GPU-Accelerated Spiking Brain (SpikingBrainGPU)
- **File:** `src/encephagen/gpu/spiking_brain_gpu.py`
- **Architecture:** All neurons in a single flat tensor. Connectivity as torch.sparse. One sparse matmul per timestep.
- **Performance:** 57x speedup over CPU NumPy (50s vs 2893s for 19K neurons, 3s simulation)
- **Per-step:** ~1.5ms per step for 19K neurons (687 steps/sec)
- **Brunel calibration:** Auto-scales j_eff with 16/C_E. Background Poisson via torch.poisson.
- **NMDA addition:** Separate slow synaptic current (tau=150ms) for PFC neurons. `i_total = i_syn + i_nmda`. NMDA receives same synaptic input but decays 30x slower.

### Spinal CPG (Matsuoka Oscillator)
- **File:** `src/encephagen/spinal/cpg.py`
- **Architecture:** Rate-based Matsuoka oscillator with 4 units (left flex/ext, right flex/ext). Mutual inhibition within leg, crossed inhibition between legs.
- **Parameters:** tau=50, tau_adapt=500, c=2.0, w_mutual=2.5, w_crossed=1.5, beta=2.5
- **Result:** 0.98 Hz alternating gait, L-R correlation = -1.000
- **Note:** Spiking CPG (adaptive LIF) failed to oscillate. Rate-based Matsuoka works because it has built-in adaptation dynamics. Pure LIF without adaptation cannot produce half-center oscillation.

### Interactive Brain Interface
- **File:** `interact.py`
- **Commands:** look A/B/C/D, sound, touch, reward, teach A, train A N, test, memory A, status, wait N, quit
- **Display:** ASCII activity bars for 12 brain regions with ▲/▼ markers
- **Learning:** Three-factor STDP (pre × post × reward) modifies connectome weights in real-time

### Body/Environment
- **SimpleBody:** MuJoCo 2-legged walker. Too stable for learning tasks (crouches to passive stability).
- **Walker2d:** Gymnasium standard bipedal walker. Properly unstable (falls in <1s). Used for Experiment 20.
- **Crawling worm:** 5-segment body. Produces forward locomotion (0.31m/10s). Abandoned in favor of human-brain focus.

## 5. What's Novel

### Genuinely new (not done before):
1. **Interactive spiking brain with human connectome topology** — nobody has built a 19K spiking brain structured by real human connectome data that you can interact with in real-time
2. **Cognitive functions emerging from connectome + identical parameters** — pattern recognition, conditioning, and working memory all operating through the actual human white matter pathways
3. **先天 × 后天 framework demonstrated** — innate structure (connectome routing produces 1.22s survival) + learned calibration (STDP improves to 1.39s) on an unstable body

### Extensions of prior work:
4. **Spiking hierarchy from connectome** (extends Gollo 2015, Zamora-Lopez 2025)
5. **NMDA-based working memory in connectome context** (extends Compte 2000)
6. **Three-factor conditioning in connectome context** (extends Izhikevich 2007)

### Not novel:
7. Pattern recognition accuracy (56%) is modest
8. Learning effects are small (+9% conditioning, +0.16s body improvement)
9. CPG-based locomotion is well-established (Ijspeert 2008)

## 6. What Failed and Why

| Experiment | What failed | Root cause |
|---|---|---|
| Exp 7-9 | Embodied learning (motor death) | Basic STDP converges to constant output — no credit assignment |
| Exp 10 | Pendulum/cart-pole RL | Surrogate gradients through recurrent spiking → high variance/exploding |
| Exp 11 | Body walking from spontaneous activity | 34-50 Hz oscillation mismatched to body's 1-2 Hz natural frequency |
| Exp 12 CPG+body | CPG locks when body feedback added | Proprioception stabilizes one CPG state, prevents alternation |
| Exp 19 | CPG calibration learning | SimpleBody too stable — crouches to passive stability, nothing to learn |
| Walking video | Body crouches, doesn't step visibly | Torques too weak to overcome gravity for full stepping |

## 9. Next Steps

### Immediate (strengthen what works):
1. **Longer training on Walker2d** — 30 episodes is short. Try 200+ episodes.
2. **Stronger learning signal** — increase learning_rate, try different reward timing
3. **Compare brain-wired vs random-wired on Walker2d** — does connectome help body control?
4. **Statistical tests** — run pattern recognition and conditioning 20 times with different seeds
5. **Web interface** — replace terminal with Streamlit/web for broader accessibility

### Medium-term (research):
6. **E-prop learning rule** — replace three-factor STDP with e-prop (Bellec 2020) for better credit assignment in motor control
7. **PopSAN approach** — population coding for motor output (Jiang 2023), known to work for bipedal walking
8. **Attractor dynamics** for working memory — go beyond NMDA decay, implement stable attractors in PFC
9. **Multiple working memory items** — can the brain hold A and B simultaneously?

### Long-term:
10. **Nengo/NEF integration** — use analytical weights for some circuits (like Spaun) combined with STDP for learning circuits
11. **Real-time performance** — optimize for interactive speed (<100ms per command)
12. **Visual system** — add hierarchical visual processing (V1→V2→V4→IT) instead of direct pattern injection
